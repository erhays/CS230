### Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?
The Gaming Room is an established video game developer that currently has a popular game in production called Draw It or Lose It. The premise of this game is that each team has one minute to guess an image that is slowly rendered on the screen. The faster the team guesses the image, the more points they are awarded. As their game is already released on Android, The Gaming Room is looking to expand their user base by releasing the game as a cross-platform distributed web app that can run on any major web browser.  

### What did you do particularly well in developing this documentation?
In developing the design document for Draw It or Lose It, I did a great job in articulating the high-level concepts related to developing a modern distributed web application. There are many modern tools and frameworks that are important to consider when deciding on a tech stack. Each decision offers trade-offs in functionality and performance. It’s critical to understand which approach is the best choice in order to satisfy the requirements set forth in the design spec. 

### What about the process of working through a design document did you find helpful when developing the code?
The Domain Model was instrumental in understanding the overall structure of the application. In it, each class was defined as well as how the individual components would interact. 

### If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?
In retrospect, I think that the Design Restraints section at the beginning of the document could use some refining. While each of the individual constraints were technically correct, I think that I missed some key constraints that were not uncovered until later in the process.

### How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?
The user’s needs evolve over time as more information is provided. That is the nature of software development. As with any creative process, in the beginning it is hard to understand what is required of the final product. It takes an iterative approach to tackle these types of problems. The best software development teams are able to constantly improve the context of the problem and uncover new solutions as the product evolves. 

### How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?
Continuous integration is one of the most valuable tools in the developer’s toolkit. As modern Agile software development practices outline, it’s important to get a working prototype into the hands of the client as quickly as possible and then improve on what we have. I think that as it relates to this project—we spent too much time working out the details when we should have just delivered something with basic functionality and then iterated on it from there.
